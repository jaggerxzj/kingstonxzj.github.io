原型模式（Prototype）

基本介绍

原型模式是指：用原型实例指定创建对象的种类，并且通过拷贝这些原型，创建新的对象。

原型模式是一种创建型设计模式，允许一个对象再创建另外一个可定制的对象，无需知道如何创建的细节。

工作原理是：通过将一个原型对象传给那个要发动创建的对象，这个要发动创建的对象通过请求原型对象拷贝它们自己来实施创建，即 对象.clone()

形象的理解：孙悟空拔出猴毛，变出其他孙悟空。

浅拷贝

克隆羊问题
现在有一只羊，姓名为tom，年龄为1，颜色为白色，创建和tom羊属性完全相同的10只羊。

代码实现：浅拷贝是使用默认的clone()方法来实现，原型类需要实现Cloneable接口

java

对于数据类型是基本数据类型的成员变量，浅拷贝会直接进行值传递，也就是将该属性值复制一份给新的对象。

对于数据类型是引用数据类型（如某个数组、某个类的对象等），浅拷贝就会进行引用传递，也就是将该成员变量的引用值（内存地址）复制一份给新对象。实例上两个对象的成员变量都指向同一个实例。

在这种情况下，在一个对象中修改该成员变量会影响到另一个对象的成员变量值。

深拷贝

深拷贝是复制对象的所有基本数据类型的变量值，为所有引用数据类型的成员变量申请存储空间，并复制每个引用数据类型成员变量所引用的对象，直到该对象可达的所有对象。也就是说，对象的深拷贝是对整个对象的拷贝

注意事项

  1. 创建新的对象比较复杂时，可以利用原型模型简化对象的创建过程，同时也能够提高效率
  2. 不要重新初始化对象，而是动态地获得对象运行地状态
  3. 如果原始对象发生改变（增加或者减少属性），其他克隆对象也会发生相应地变化，无需修改代码
  4. 在实现深克隆的时候可能需要比较复杂的代码
  5. 缺点： 需要为每一个类配备一个克隆方法，这对全新的类来说不是很难，但对已有的类进行改造时，需要修改其源代码，违背了ocp原则。
